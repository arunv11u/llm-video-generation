# MVP: Photorealistic Virtual-Influencer Reel Generator (v3)

## Context

**Goal:** Build an MVP that produces glamorous, Instagram-ready (9:16, ~15s) vlog reels of a **single consistent photorealistic woman** who **silently mouths the words of a transcript** the user provides, while a **background music track** plays as the final audio.

**Why this revision:** Research uncovered [SkyReels V3 A2V 19B](https://huggingface.co/Skywork/SkyReels-V3-A2V-19B) (released Jan 2026) — a single model that takes **one portrait + one audio file** and produces a realistic talking-head video with **native lip-sync, head motion, blinks, and micro-expressions**. This eliminates the need for separate lip-sync models (LatentSync, MuseTalk, Wav2Lip) entirely. The pipeline collapses from 6 steps / 5 models to **4 steps / 3 models**.

**Key interaction:**
- **Input:** `--transcript "hey guys, welcome back..."` + `--music bg.mp3` + optional scene prompt
- **Final MP4 audio:** the user's background music only
- **What the viewer sees:** glamorous woman naturally mouthing the transcript words with realistic head motion, over music — a fashion/vlog reel aesthetic

**Pipeline overview (dramatically simplified):**

```
transcript
    │
    ▼
F5-TTS  →  throwaway tts_voice.wav  (never heard by viewer)
    │
    ▼
character/reference.png + tts_voice.wav
    │
    ▼
SkyReels V3 A2V 19B  →  video with native lip-sync + head motion + expressions
    │                    (one model does EVERYTHING — no overlay, no compositing)
    ▼
ffmpeg: swap TTS audio → user's bg.mp3, burn transcript captions, 9:16 crop
    │
    ▼
outputs/reel_<ts>.mp4  (music + lip-synced visuals, ready to post)
```

**Why SkyReels V3 A2V is the right choice:**
- Lip-sync is **native to video generation** — not painted on as a post-process overlay. No mouth blur, no composite seams, no two-model conflict.
- Natural head tilts, blinks, eyebrow raises that match speech rhythm — not a static portrait.
- 40+ language lip-sync support.
- 19B Diffusion Transformer — highest parameter count in open-source talking-head.
- Skywork License (proprietary but free to download and use — weights are on HuggingFace, no payment required).
- Sources: [GitHub](https://github.com/SkyworkAI/SkyReels-V3), [HuggingFace](https://huggingface.co/Skywork/SkyReels-V3-A2V-19B), [WaveSpeed blog](https://wavespeed.ai/blog/posts/introducing-wavespeed-ai-skyreels-v3-talking-avatar-on-wavespeedai/)

**What this is NOT:** Not an LLM-script-to-video product. The user writes the transcript. The heavy lifting is image generation + TTS + audio-driven video synthesis.

**Why H100 80GB:** SkyReels V3 A2V is 19B parameters. Needs ~60–80 GB VRAM at full quality. RunPod H100 80GB (~$2.50–4/hr).

## Product shape (MVP)

Two stages — CLI pipeline + Gradio UI:

### 1. Character Studio (run once)
- Generate ~4 candidate photorealistic portraits with Flux.1-schnell + PuLID.
- User picks one → `character/reference.png`.
- This single portrait is reused for every future reel (virtual-influencer brand).

### 2. Reel Generator (run per video)

**Inputs:**
- `--transcript "hey guys, welcome back..."` (words she'll mouth)
- `--music path/to/bg.mp3` (final audio the viewer hears)
- `--prompt "selfie vlog, golden hour, Parisian cafe"` (passed to SkyReels V3 A2V as its text prompt — controls expression, mood, and shot style, e.g. "A woman is giving a speech. She is confident, poised, and joyful. Use a static shot.")

**Steps:**

| Step | What happens | Model | Output |
|------|-------------|-------|--------|
| A | Convert transcript to speech | **F5-TTS** | `tts_voice.wav` (~15s, throwaway) |
| B | Generate talking-head video | **SkyReels V3 A2V 19B** | `raw_video.mp4` — portrait comes alive: lip-sync + head motion + expressions, driven by TTS audio |
| C | Audio swap + caption burn | **ffmpeg** | Swap TTS audio → user's music (trimmed to video length, 0.5s fade-out). Burn transcript as captions (evenly timed across video duration; or use faster-whisper on `tts_voice.wav` for word-level timing if needed). Export final MP4. |

**Output:** `outputs/reel_<timestamp>.mp4` — ~15 seconds, 720p native (9:16). Instagram compresses to ~720p on most devices anyway. Upscale to 1080×1920 is deferred to M2 if needed.

**Note on scene/background:** SkyReels V3 A2V takes a portrait + audio + text prompt. The text prompt controls expression and shot style. The background comes from the portrait or is generated by the model. For specific scenes (café, beach, etc.), generate anchor portraits with the desired background using Flux+PuLID, and use the SkyReels prompt to control mood/expression.

## Model stack (as of April 2026)

| Stage | Model | License | VRAM | Why this one |
|-------|-------|---------|------|-------------|
| Anchor portrait | **Flux.1-schnell** | Apache-2.0 | ~8 GB | Gold standard photorealistic stills. Commercial-safe. Generates the portrait that SkyReels V3 will animate. |
| Face identity lock | **PuLID** at 0.8 | Apache-2.0 | ~2 GB | Locks face to `reference.png` when generating new anchor portraits (different scenes/outfits) after M0. Not used in M0 — M0 generates free-form candidates with Flux alone. |
| TTS (throwaway) | **F5-TTS** | Apache-2.0 | ~4 GB | SOTA open-source TTS. Converts transcript to audio that drives SkyReels. Never heard by viewer. Runs as a Python library call on the pod (`f5_tts` pip package). |
| **Video generation + lip-sync (unified)** | **SkyReels V3 A2V 19B** | Skywork License (free, proprietary) | ~60–80 GB | **The core model.** Portrait + audio + prompt → realistic talking-head video. Native lip-sync, head motion, blinks, micro-expressions. No separate lip-sync model needed. Runs via its Python CLI (`generate_video.py --task_type talking_avatar`). |
| Audio swap + captions | **ffmpeg** | LGPL | CPU only | Swap audio tracks, burn captions, export. Runs on the pod. |
| Captions | Transcript text → ffmpeg `drawtext`/`subtitles` | — | — | Evenly space words across video duration for MVP. If finer timing needed, run faster-whisper once on the throwaway `tts_voice.wav`. |

**Fallbacks (only if SkyReels V3 A2V fails):**
- **SkyReels V2 14B I2V + LatentSync v1.6** — the previous plan's architecture. More complex (5 models, overlay conflict risk) but proven separately.
- **Wan 2.2 14B I2V + LatentSync** — if SkyReels ecosystem is problematic. Mature ComfyUI support.
- **VideoReTalking** — if we get good SkyReels V2 video but need to fix the mouth only. Takes video in, replaces only the mouth region while preserving everything else.

## Risks (dramatically reduced from previous plan)

1. **SkyReels V3 ComfyUI maturity (HIGHEST RISK)** — V3 is 3 months old (Jan 2026). ComfyUI support may be incomplete. **Mitigation:** bypass ComfyUI entirely for the V3 step — call SkyReels V3 via its Python API / CLI directly. Only use ComfyUI for the Flux portrait generation step (which is mature). This is actually simpler.

2. **SkyReels V3 A2V face identity preservation** — how well does V3 A2V preserve the face from the input portrait? The model page says it works from "one portrait photo" but identity drift over 15s is possible. **Validate in M1.** If drift is severe, M2's Flux LoRA will help with the anchor, but V3 A2V itself would need a character LoRA (check if SkyReels V3 supports LoRA fine-tuning).

3. **TTS prosody → mouth naturalness** — F5-TTS drives the lip-sync. Flat prosody = mechanical mouth. Fix: F5-TTS expressive mode. Less risky than the old plan because V3 A2V was specifically trained on audio-visual alignment (including the SkyReels-Audio research from June 2025), so it handles TTS audio better than LatentSync did.

4. **Scene/background control** — SkyReels V3 A2V does take a text prompt, but it controls expression/mood/shot style, not detailed scene composition. Background comes mostly from the portrait. For scene variety, generate different Flux anchor portraits (same face via PuLID, different scenes).

5. **Hand artifacts** — if hands are visible in the portrait, V3 may produce artifacts. **Mitigation:** frame the anchor portrait as a close-up/medium shot (shoulders up) to avoid hands.

6. **Silent mouthing UX** — viewers see clear lip movement but hear only music. Intentional aesthetic (fashion/vlog reels), but validate it resonates with the target audience on the first 1–2 test reels.

7. **Music licensing** — pipeline doesn't touch this, but if the user's `bg.mp3` is copyrighted, posting to IG requires a license. README flags this.

8. **Model licenses** — Flux.1-schnell (Apache-2.0), F5-TTS (Apache-2.0), PuLID (Apache-2.0). SkyReels V3 uses Skywork License (proprietary but free to download and run; no payment required for weights).

9. **H100 cost** — ~$0.30–0.80 per reel in steady state. Fine for monetized content. At scale (100+ reels/day), consider a reserved H100.

**Risks ELIMINATED vs. previous plan:**
- ~~LatentSync overlay conflict with video generator~~ — no overlay; lip-sync is native
- ~~Mouth-region blur/artifacts from lip-sync model~~ — no separate lip-sync model
- ~~SkyReels V2 + LatentSync two-model interaction~~ — single model now
- ~~ComfyUI node complexity (6 workflows)~~ — only 1 ComfyUI workflow (portraits); V3 called directly

## Verification plan

**Milestone 0 — Pick the face:**
1. Pod running, `COMFY_HOST` set.
2. `python pipeline/pick_portrait.py --prompt "..." --count 4`
3. Success: 4 photorealistic glamorous portraits in `outputs/candidates/`.

**Milestone 1 — Full reel (THE gate):**
1. `character/reference.png` set. Any royalty-free ~15s background music clip ready.
2. Run `python pipeline/run_reel.py --transcript "..." --music bg.mp3`
3. Success criteria:
   - Output is a 720p 9:16 MP4, duration matches TTS speech (~8–15s).
   - **Audio is the user's music** (verify with `ffprobe`; no TTS voice audible).
   - Woman has natural head motion, blinks, expressions — not a static portrait.
   - **Lip movement clearly matches the transcript words** (eyeball test: can you read the words from her mouth when captions appear?).
   - Face is recognizably `reference.png` throughout.
   - Burned captions match transcript exactly, timed to mouth.
   - Clean audio: no desync, no pops, smooth 0.5s fade-out.
4. **If this passes → MVP is done.** Ship it, start making reels.
5. **If face drifts → investigate V3 LoRA or fall back to V2+LatentSync.**
6. **If V3 integration fails entirely → fall back to previous plan (V2+LatentSync), which is already designed and documented.**
